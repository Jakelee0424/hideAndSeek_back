## 진행 단계 — 설계 요점

- **전이는 "시작 시각 + 경과"로 매번 계산**한다(PhaseTimeline). tick이 밀려 한 번에 두 단계를 건너뛰어도
  상태가 안 어긋난다. 이벤트로 쌓지 않는 이유.
- **단계는 매 tick 안 싣는다.** 로스터와 같은 규약으로 전환 시·입장 시에만 보내고, 카운트다운은 클라가
  남은 시간(phaseRemainMs)에서 자체 진행. 실측 스냅샷 497개 중 5개만 phase 포함.
- 길이는 `game.phases.*`(Duration). 데모용 단축: `--game.phases.mission=5s` 식으로 통째로 덮어쓴다.
- **이중 관리**: 프론트 `net/types.ts`의 `GamePhase`/`PHASE_LABEL` ↔ 백 `game/GamePhase.java`.

### 🔶 알려진 한계 — "게임 시작" 신호가 없다

시계가 **방의 첫 tick(=첫 사람 입장)부터** 흐른다. 대기방에 있는 동안에도 온보딩이 깎인다.

## AI 봇 — 이슈 1 해결의 핵심

원인이 **셋**이었다(노트엔 둘로 적혀 있었는데 진짜 범인은 프롬프트였다).

1. **프롬프트가 시키고 있었다.** SYSTEM의 `아직 아무것도 안 풀렸으면 힌트부터 확인` 규칙은, 아무도
   퍼즐을 안 푸는 동안 solved가 영영 비어 있어 6초마다 무조건 발동한다. 모델이 무시한 게 아니라
   시킨 대로 한 것. → 조건을 "쪽지를 안 읽었으면"으로.
2. **봇에게 기억이 없었다.** lastArrivedId(직전 하나)로는 "쪽지 읽음"을 못 담는다. → visited(방문 전체).
3. **스크립트 층의 별개 왕복.** nearestUnsolved가 직전 하나만 제외 → 해결가능 지점이 둘뿐이라 왕복 확정.
   → 다녀온 곳 전부 제외.

실측: LLM 계획 GOTO_NOTE **10/10 → 1/5**, 궤적 lockbox↔door 8회 왕복 → 각 1회 후 사람에게 붙음,
사람 최소 접근 9.81m → 0.09m(이슈 2도 해소).

💡 **교훈**: 모델이 이상하게 굴면 프롬프트를 먼저 의심할 것. 규칙의 발동 조건이 항상 참인지 확인하기
전에 "모델이 무시한다"고 단정하지 말 것.

## Groq 토큰 — 이번에 확정한 사실

- **한도는 모델별로 따로, 사용량은 모델·프롬프트마다 다르다.** `max_tokens`는 상한일 뿐 예약이 아니다
  (실제 한도/과금은 쓴 만큼만). 올리는 건 사실상 공짜.
- gpt-oss는 답 앞에 **추론 토큰**을 쓴다. 실제 답(JSON)은 30~40토큰, 나머지 200~440이 추론.
  프롬프트를 규칙 3줄→5줄+visitedIds로 늘리자 completion이 110~118 → 207~365로 3배가 되며 400 상한에
  닿아 json_validate_failed 재발. `max_tokens=1000`으로 0/6 실패.
- ⚠️ **프롬프트 압축은 역효과였다**(규칙 1줄로 뭉치니 6/6 실패). 짧다고 추론이 주는 게 아니다.
- ⚠️ 프롬프트를 건드리면 추론 토큰이 같이 움직인다. **늘린 뒤엔 실패율을 반드시 다시 잴 것.**

## 실행 / 검증 (이 세션 실측)

```powershell
# 백엔드 — JAVA_HOME을 21로. 기본 java가 8이라 -Dorg.gradle.java.home만으론 Gradle 런처가 못 뜬다.
$env:JAVA_HOME = "C:\Users\onnet-user\.jdks\ms-21.0.11"
$env:GROQ_API_KEY = [Environment]::GetEnvironmentVariable('GROQ_API_KEY','User')
.\gradlew bootRun --no-daemon "--args=--server.port=8081 --game.bot.llm.enabled=true --logging.level.com.game3d.server.game=DEBUG"

# 프론트 — 반드시 :3000 (백엔드 allowed-origins가 localhost:3000, 다른 포트는 WS 403)
$env:NEXT_PUBLIC_WS_URL = "ws://localhost:8081/ws"
.\node_modules\.bin\next dev -p 3000   # npm run dev -- --port 는 PowerShell에서 --가 씹힌다
```

- ⚠️ `--no-daemon` 필수(데몬이 키 설정 전 환경을 물려받으면 401).
- ⚠️ 8080엔 사용자 개발 서버가 뜰 수 있다. 죽이지 말고 8081.
- ⚠️ 프론트 `node_modules` 없으면 `npm ci` 먼저.
- 봇 검증: `node --experimental-websocket scripts/botcheck.mjs 8081`
  판별 원리: 스크립트 층은 note-1에 안 가므로 봇의 note-1 도달 = LLM 동작 증거.
- 단계 검증(시간 단축): `--game.phases.onboarding=5s --game.phases.mission=5s --game.phases.sharing=5s --game.phases.vote=5s`
